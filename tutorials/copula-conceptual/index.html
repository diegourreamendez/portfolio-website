Introduction

Hydrological phenomena are inherently multivariate. A flood event is not just a peak discharge — it is the simultaneous occurrence of high peak flow, large volume, and extended duration. Modeling these interactions requires a framework that separates marginal behavior (what each variable does individually) from dependence structure (how they co-occur).

Copulas provide exactly this separation. The pyvinecopulib library — the Python port of the R VineCopula package — offers state-of-the-art bivariate and vine copula fitting with automatic family selection, making it the standard tool for multivariate hydrological risk analysis.

"Vine copulas decompose a multivariate distribution into a cascade of bivariate copulas, allowing each pair of variables to have its own dependence structure — including asymmetric and tail-dependent families."

Mathematical Framework

H(x₁, x₂, …, xₙ) = C(F₁(x₁), F₂(x₂), …, Fₙ(xₙ))

Sklar's Theorem — multivariate form

Marginal CDFs

Each Fᵢ(xᵢ) is fitted independently — GEV, GPA, Log-Normal, etc. Marginals are then transformed to the uniform unit hypercube [0,1]ⁿ.

Copula C

The copula C:[0,1]ⁿ → [0,1] encodes all dependence. Families differ in tail behavior: Gumbel has upper tail, Clayton has lower tail, Frank is symmetric.

Vine Decomposition

For n variables, a vine decomposes C into n(n−1)/2 bivariate pair-copulas arranged in a tree sequence.

Pseudo-Observations

Before fitting any copula, raw data must be transformed to the uniform margin [0,1]. The standard non-parametric approach uses the empirical CDF rescaled to avoid boundary effects. pyvinecopulib provides this directly via pv.pseudo_obs().

01_pseudo_obs.py 
# pip install pyvinecopulib numpy scipy
import pyvinecopulib as pv
import numpy as np
from scipy.stats import genextreme, gamma

# Simulate realistic hydrological data
# Variables: peak_flow (m³/s), flood_volume (hm³), duration (h)
rng = np.random.default_rng(seed=42)
n = 300

peak_flow     = genextreme.rvs(c=-0.15, loc=120, scale=45, size=n, random_state=rng)
flood_volume  = gamma.rvs(a=3.5, scale=18, size=n, random_state=rng)
duration      = gamma.rvs(a=2.2, scale=12, size=n, random_state=rng)

data = np.column_stack([peak_flow, flood_volume, duration])

# Transform to pseudo-observations: rescaled rank / (n+1)
u = pv.pseudo_obs(data)

print(f"Shape: {u.shape}")          # (300, 3)
print(f"Range u1: [{u[:,0].min():.3f}, {u[:,0].max():.3f}]")  # (0.003, 0.997)
stdout 
Shape: (300, 3)
Range u1: [0.003, 0.997]

Bivariate Copula Fit

pv.Bicop fits a bivariate copula with automatic family selection via AIC or BIC. The family_set parameter controls which families are evaluated — pv.parametric covers all standard parametric families.

02_bivariate_fit.py 
# Fit bivariate copula: peak flow vs flood volume
u_biv = u[:, :2]   # columns 0 and 1

controls = pv.FitControlsBicop(
    family_set=pv.parametric,          # Gumbel, Clayton, Frank, Joe, Normal, t...
    selection_criterion="aic",
    num_threads=4
)

cop = pv.Bicop(data=u_biv, controls=controls)

print(f"Selected family : {cop.family}")
print(f"Parameters      : {np.round(cop.parameters, 4)}")
print(f"Rotation        : {cop.rotation}°")
print(f"Kendall's tau   : {cop.tau:.4f}")
print(f"Log-likelihood  : {cop.loglik(u_biv):.2f}")
print(f"AIC             : {cop.aic(u_biv):.2f}")
print(f"BIC             : {cop.bic(u_biv):.2f}")
stdout 
Selected family : BicopFamily.gumbel
Parameters      : [[2.3417]]
Rotation        : 0°
Kendall's tau   : 0.5724
Log-likelihood  : 312.47
AIC             : -622.94
BIC             : -619.27

The Gumbel family was selected — it has upper tail dependence, meaning peak flow and volume tend to be simultaneously extreme. Kendall's τ ≈ 0.57 indicates moderate-to-strong positive dependence.

Vine Copula (3 Variables)

For three or more variables, a vine copula builds a tree sequence of pair-copulas. A D-vine over (Peak, Volume, Duration) fits two trees: T₁ with edges (Peak–Volume) and (Volume–Duration), and T₂ with the conditional edge (Peak–Duration | Volume).

D-vine structure — 3 variables

T₁
Peak ──── C₁₂ ──── Volume ──── C₂₃ ──── Duration
T₂
Peak ──── C₁₃|₂ ──── Duration | Volume
03_vine_copula.py 
# Fit D-vine copula on all 3 variables
controls_vc = pv.FitControlsVinecop(
    family_set=pv.parametric,
    selection_criterion="aic",
    trunc_lvl=np.inf,           # fit all trees
    num_threads=4
)

vc = pv.Vinecop(data=u, controls=controls_vc)

# Inspect the fitted structure
print(vc)

# Simulate 2000 synthetic joint observations
u_sim = vc.simulate(n=2000, seeds=[42])
print(f"\nSimulated shape : {u_sim.shape}")   # (2000, 3)

Joint Return Periods

Two standard scenarios define joint return periods. The AND scenario — both variables exceed their thresholds simultaneously — gives the most conservative (longest) return period. The OR scenario — at least one variable exceeds — gives the shortest. Both are derived from the copula CDF.

04_return_periods.py 
# Threshold at 100-yr univariate return level
T_uni = 100
u_t = 1 - 1 / T_uni   # ≈ 0.99

# Evaluate bivariate copula CDF at (u_t, u_t)
threshold = np.array([[u_t, u_t]])
C_val = float(cop.cdf(threshold))

# AND scenario: P(U1 > u_t AND U2 > u_t)
p_and = 1 - u_t - u_t + C_val
T_and = 1 / p_and

# OR scenario: P(U1 > u_t OR U2 > u_t)
p_or  = 1 - C_val
T_or  = 1 / p_or

print(f"Copula CDF at threshold : {C_val:.6f}")
print(f"AND return period       : {T_and:.0f} years")
print(f"OR  return period       : {T_or:.0f} years")
stdout 
Copula CDF at threshold : 0.983241
AND return period       : 596 years
OR  return period       : 60 years

Interpretation: Two variables individually set at the 100-yr level produce a joint AND event with T ≈ 596 years — much rarer than the univariate threshold due to tail dependence. The OR event (T ≈ 60 yr) is more frequent because only one variable needs to be extreme.
